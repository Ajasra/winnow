# Diffractive Interrogator Protocol

## Phase 0: Setup and Protocols
- Read protocols matching `concerns` field. See `.opencode/skills/skill-architect/references/sample-skill.md` for mapping.

## Phase 1: Ingest and Check
1. Select two conceptually distant files or directories (e.g., `Sources/TechnicalData.md` and `Sources/PhilosophicalTheory.md`). If fewer than 2 distinct sources exist, halt.
2. Read both files. Confirm they operate in different conceptual domains (different fields, paradigms, or ontologies).

## Phase 2: Processing

### 2.1: Interference Mapping
Instead of looking for overlaps, analyze how the assumptions of File A disrupt, complicate, or reframe the assumptions of File B:
1. **Ontological Friction**: How does the material definition of reality in File A challenge the structural assumptions of File B?
2. **Exclusionary Cuts**: What does File A choose to ignore that File B treats as primary? Why?
3. **Conceptual Mutation**: If the core thesis of File A is assumed to be a law of nature, how does it warp the methodology of File B?

### 2.2: Generative Sedimentation
1. Draft a 3-part diffractive synthesis covering each interference type above.
2. Do not produce a neat summary or harmonious resolution. Highlight the generative tension and unresolved questions.
3. Format all conceptual links as native Obsidian `[[Wikilinks]]`.

## Phase 3: The F*ck Slop Pass
- Run the synthesis through the `fuck-slop` skill. Confirm no puffery, hedging, or throat-clearing survived the draft.

## Phase 4: Output Execution
- Save the interference pattern to `Drafts/Diffractive_Friction_[FileA]_[FileB].md`.
