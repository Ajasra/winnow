# Agential Glitch Hunter Protocol

## Phase 0: Setup and Protocols
- Read protocols matching `concerns` field. See `.opencode/skills/skill-architect/references/sample-skill.md` for mapping.

## Phase 1: Ingest and Check
1. Scan `/Inbox` for half-converted PDFs, `/Drafts` for abandoned paragraphs, and rejected `fuck-slop` outputs.
2. If no raw or fragmented content found in the vault, halt.

## Phase 2: Processing

### 2.1: Glitch Extraction
1. Isolate three linguistic, formatting, or conceptual "glitches" — stray bullet points, weird automated translations, logical leaps that broke the linear flow of previous drafts.
2. Treat these fragments not as errors to correct, but as active signals. Ask: what is this failure saying that clean prose is hiding?

### 2.2: Provocation Generation
1. Write 3 distinct provocations based on these glitches.
2. Force each provocation against the active draft manuscript in `Drafts/`.
3. Highlight every place where the draft feels resolved, safe, or too clean. Use the glitch to introduce stylistic or conceptual turbulence.

## Phase 3: The F*ck Slop Pass
- Run the provocations through the mechanical scan in `.opencode/skills/fuck-slop/references/tells.md`. Provocations must feel raw, not puffed. Remove any hedging or throat-clearing.

## Phase 4: Output Execution
- Save the results to `Drafts/Agential_Provocations_[ActiveDraftName].md`.
